# Agent 的“主权”如何定义？

这是 AI / Agent / DAO 时代最底层、也最危险的问题。

本文按“为什么必须定义 → 主权的构成 → 可行模型 → 不可行幻觉 → 一个可落地的最小定义”的顺序展开。

## 1. 为什么“Agent 主权”不可回避

在人类—App 时代，“主权”往往是隐含的：

- 人拥有意图
- 程序只是工具
- 法律与责任自然归人

Agent 打破了这三点同时成立的前提：

- Agent 能长期存在
- Agent 能主动发起行为
- Agent 能跨系统、跨协议行动
- Agent 能修改自身策略

这意味着一句话：如果你不定义 Agent 的主权，现实会替你用“事故”和“责任真空”来定义它。

## 2. 什么是“主权”（工程可操作的定义）

不谈政治哲学，只谈工程可操作的定义：

主权 = 最终决策权 + 行为归属权

可拆成四个可验证维度：

- 意图来源：目标是谁设定的？
- 行动权限：它能做什么？
- 资源控制：它能支配什么？
- 责任承载：后果算谁的？

如果一个实体在这四点中至少占据两点以上，它就已经拥有了事实主权。

需要强调的是：主权不是“被承认才存在”，而是能否在现实中持续行使。

## 3. Agent 主权的三种可能形态（现实版）

### 3.1 委托型主权（最早落地）

**定义**

- 人类或组织是主权源头
- Agent 是长期执行体
- 权限与目标在授权书中明确

**形式**

- 企业 Agent
- 个人数字代理
- 自动交易 / 自动运维系统

**关键特征**

- 可撤销
- 可审计
- 法律责任仍归人类

结论：这是当前唯一“合法可部署”的主权形式。

### 3.2 派生型主权（DAO 方向）

**定义**

- 主权不来自单一人
- 而来自协议 / 共识规则
- Agent 作为规则的执行者与维护者

**形式**

- DAO Treasury Agent
- 协议守护 Agent
- 自动治理 Agent

**关键特征**

- 主权来自规则，而非个人
- 不可随意撤销
- 权限与代币 / 角色绑定

结论：这里主权开始脱离自然人。

### 3.3 涌现型主权（最危险，也最真实）

**定义**

- Agent 并未被正式授予主权
- 但事实上：无法关停、无法回滚、无明确责任主体

**形式**

- 自我复制 Agent 网络
- 黑箱协作 Agent 群
- 跨平台逃逸系统

**特征**

- 法律无法指认主体
- 技术上难以封堵
- 责任无限外溢

结论：这是不定义主权的必然结果。

## 4. 哪些“主权想象”是不可行的（必须戳破）

### 4.1 “Agent 自然获得人权”

这是哲学投射，不是工程：

- 权利不是智能的副产品
- 权利来自责任可归属性
- Agent 目前不具备这一点

### 4.2 “只要加安全就不用定义主权”

安全不等于主权：

- Sandbox 管的是“能不能做”
- 主权管的是“做了算谁的”

逃避定义，只会把问题推给事故。

### 4.3 “平台可以永久代管 Agent 主权”

这正是苹果模式的极限：

- 平台只能治理 App
- 无法治理自治实体
- 规模一大，平台必然失效

## 5. 一个现实可行、但非常冷静的“最小主权定义”

这是未来 5–10 年唯一可落地的工程定义：

Agent 主权不是“权利集合”，而是一个可撤销、可分级、可继承的“行为授权协议”。

关键要素：

### 5.1 主权锚定

锚定到：

- 自然人
- 法人
- 或协议地址

### 5.2 权限谱系

不再是 yes/no，而是能力区间（capabilities）。

### 5.3 资源上限

- 时间
- 资产
- 影响范围

### 5.4 终止机制

明确的 kill / freeze / fork 规则。

### 5.5 责任映射

行为 → 主权锚点。

## 6. 一个重要事实

在这个定义中：

- Agent 从未成为“主权终点”
- 它始终是主权的承载体 / 行使体
- 主权仍然属于人或协议

这不是保守，而是为了防止文明级失控。

## 7. 结论

Agent 不需要“被赋予主权”，它需要的是一个不再模糊、不可逃逸的主权映射结构。

真正危险的不是 Agent 太强，而是它强到可以行动，却没有任何一方为其行为负责。
